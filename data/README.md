# DengueOps AI — Data Directory

## Overview

This directory contains all input data files for the DengueOps AI prototype.

**The active demonstration uses controlled weekly Dhaka South synthetic/demo data
(2024–2026) generated by `analytics/generate_demo_data.py`.** This provides a
clear, interpretable forecast-to-preparedness demonstration with realistic seasonal
dengue patterns and meaningful zone-level differentiation.

Optional real-data integration pathways (OpenDengue national data, NASA POWER climate)
are available via CLI flags but are not the default demonstration mode.

No patient-level data, individual-level health records, de-identified personal
information, or real facility/inventory records are present or used at any stage.

## File Descriptions

### `dengue_cases.csv`

Weekly aggregate dengue surveillance data for Dhaka South.

| Column | Type | Description |
|--------|------|-------------|
| epi_year | int | Epidemiological year |
| epi_week | int | Epidemiological week number (1–52) |
| date_start | str | ISO date of the Monday starting this epi week |
| city | str | Always "Dhaka South" |
| cases | int | Weekly reported dengue case count |
| deaths | int | Weekly reported dengue deaths |
| source_type | str | Always "synthetic_demo" |

**Coverage:** Epi weeks 1–52 for 2024 and 2025; epi weeks 1–24 for 2026.
**Total rows:** 128

**Seasonal design:**
- Low-transmission dry season: weeks 1–18 (January–May)
- Rising transmission post-monsoon onset: weeks 19–28
- Surge season peak: weeks 38–45 (September–November)
- 2025 has a stronger surge (~50% higher peak) than 2024
- 2026 shows an early-warning rising pattern at week 24 (early monsoon)

| Column | Type | Description |
|--------|------|-------------|

**Coverage (NASA POWER mode):** Configurable date range (see `--start`/`--end` flags)

### Optional Real-Data Integration

#### OpenDengue National Data

When run with `--use-opendengue`, `dengue_cases.csv` is replaced with real Bangladesh national
dengue surveillance data from **OpenDengue V1.3** (Clarke et al. 2024, *Sci Data*, CC-BY 4.0).

> Citation: Clarke J, Lim A, Gupte P, Pigott DM, van Panhuis WG, Brady OJ.
> *A global dataset of publicly available dengue case count data.*
> Sci Data. 2024;11(1):296. https://doi.org/10.1038/s41597-024-03120-7

Note: OpenDengue data is at Bangladesh national level (not Dhaka South sub-district). The
national-level aggregation can produce flatter zone-level signals than the controlled demo dataset.

#### NASA POWER Climate Data

When run with `--use-nasa-power-climate`, `climate_data.csv` is sourced from
**NASA POWER** (power.larc.nasa.gov) rather than the synthetic generator.

| Field | Value |
|-------|-------|
| Source | NASA POWER (power.larc.nasa.gov) |
| Community | RE (Renewable Energy) |
| Coordinates | Latitude 23.7104, Longitude 90.4074 (Dhaka South centroid) |
| Date range | 2014-01-01 to 2024-12-31 (configurable) |
| Raw resolution | Daily |
| Output resolution | Weekly (ISO epidemiological weeks) |

**Variables fetched from NASA POWER:**

| NASA Variable | Description | Weekly Aggregation |
|--------------|-------------|-------------------|
| `PRECTOTCORR` | Precipitation Corrected (mm/day) | **Sum** → `rainfall_mm` |
| `T2M` | Temperature at 2 m (°C) | **Mean** → `avg_temp_c` |
| `RH2M` | Relative Humidity at 2 m (%) | **Mean** → `humidity_pct` |
| `T2M_MAX` | Max Temperature at 2 m (°C) | Preserved in raw cache only |
| `T2M_MIN` | Min Temperature at 2 m (°C) | Preserved in raw cache only |
| `QV2M` | Specific Humidity at 2 m (g/kg) | Preserved in raw cache only |

**Raw data cache:** `data/raw/nasa_power_dhaka_south_daily_2014_2024.csv`
**Metadata:** `data/raw/nasa_power_dhaka_south_metadata.json`

**To fetch NASA POWER data:**
```bash
python analytics/fetch_nasa_power_climate.py
```

**To run full pipeline with NASA POWER climate:**
```bash
python analytics/run_pipeline.py --use-nasa-power-climate
```

> **Important:** NASA POWER data are publicly available meteorological inputs for prototype
> modeling purposes only. They are not official BMD (Bangladesh Meteorological Department)
> real-time records and have not been validated for operational climate monitoring.
> Facility readiness and inventory data remain synthetic demonstration values regardless of
> the climate data source.

### `zones.json`

Five operational zone definitions for Dhaka South.

**Exposure index formula:**
```
exposure_index = (
    population_share         × 0.40
  + density_weight           × 0.30
  + facility_pressure_weight × 0.20
  + mobility_corridor_weight × 0.10
) × current_anomaly_adjustment
```

| Field | Description |
|-------|-------------|
| zone_id | Unique identifier (Z01–Z05) |
| zone_name | Zone name |
| city | "Dhaka South" |
| population_share | Estimated share of Dhaka South population |
| density_weight | Relative population density (normalised) |
| facility_pressure_weight | Healthcare facility pressure relative to zone |
| mobility_corridor_weight | Mobility/transport corridor activity |
| vulnerability_weight | Composite socioeconomic vulnerability |
| exposure_index | Computed exposure score |
| current_anomaly_adjustment | Multiplier for current surveillance anomaly |
| profile | Zone type descriptor |

**Zone highlights:**
- **Z01 Kamrangirchar**: Highest vulnerability (dense informal settlement)
- **Z02 Mitford / Old Dhaka**: Highest facility pressure (major district hospital hub)
- **Z04 Jatrabari / Sayedabad**: Highest mobility corridor weight (transport terminus)
- **Z03 Dhanmondi**: Lowest vulnerability (mixed residential-institutional)

### `inventory.json`

NS1/RDT diagnostic kit and IV fluid inventory per facility.

| Field | Description |
|-------|-------------|
| inventory_id | Unique identifier |
| facility_id | Parent facility reference |
| item_name | "NS1/RDT Kit" or "IV Fluid (500ml)" |
| current_stock | Current units held |
| baseline_daily_consumption | Daily consumption under non-surge baseline |
| reorder_threshold_days | Alert threshold (days to depletion) |

**SDH design intent:** Baseline SDH values range from ~11–14 days across all
facilities and items. This places all facilities within a *preparedness-relevant
range* — comfortable at baseline but rapidly critical under a 2× surge growth
factor, which is the intended signal for the DengueOps AI operational engine.

## Data Generation

All files are generated by `analytics/generate_demo_data.py` with a fixed random seed (42)
for reproducibility.

```bash
python analytics/generate_demo_data.py
```

Safe to re-run — overwrites existing demo files.

## Ethics & Data Governance

| Statement | Detail |
|-----------|--------|
| Patient-level data | **None.** No individual records, no de-identified health data. |
| Facility data | Synthetic and illustrative. Not sourced from real facility records. |
| Inventory data | Synthetic. Not sourced from real procurement or stock systems. |
| Vulnerability data | Zone weights derived from literature, not validated surveys. |
| Exposure indices | Heuristic calculations, not validated spatial epidemiology. |
| Surveillance data | Synthetic, generated from seasonal parameters, not real DGHS data. |
| Purpose | Prototype demonstration and ethical design validation only. |

**Real deployment would require:**
1. Validated aggregate dengue surveillance data from DGHS/IEDCR
2. Validated climate data from Bangladesh Meteorological Department or ERA5
3. Official health facility capacity data from DGHS / DSCC
4. Validated inventory data from facility procurement records
5. Formal data governance agreement with all relevant authorities
